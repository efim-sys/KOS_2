# Система для Korobochka v2 и v3

**Основные компоненты:**

- /programs - исходный код для standalone приложений, загружаемых как .elf
- /src/KUI - графический интерфейс для отображения меню
- /src/KOS - системные функции для работы с периферией, загрузка изображений и т.п.
- /src/korobochka_commander - файловый менеджер для microSD
- /src/launcher - загрузчик .elf файлов
- другие папки в /src - встроенные приложения
- /src/define.h - настройки для разных экранов и версий платы
- /src/main.cpp - основной файл, с которого начинается работа
- /data - файлы, которые будут помещены во внутреннюю SPIFFS

**Установка IDF**

Сейчас у меня основная прошивка написана под фреймворк Platformio, который является расширением для visual studio code. Загружать и собирать придется через него. В visual studio code слева в разделе extensions нужно будет скачать PlatformIO IDE

Если открыть в vs code каталог с этим репозиторием, по конфигу platformio.ini начнется загрузка 2 Гб фреймворка esp-idf, где содержится исходный код операционки freertos, компиляторы для всех моделей esp.

Текущего пользователя нужно добавить в группу dialout для доступа к последовательному порту

```
sudo usermod -a -G dialout <username>
```

**Загрузка и отладка**

Компилируется и загружается вся прошивка целиком по кнопке стрелочка в нижней части окна.

Основной способ отладки - через последовательный интерфейс. В vs code он под кнопкой вилки в нижней части окна, или можно через терминал:

```
tio --map INLCRNL /dev/ttyACM0
```

Порт может быть /dev/ttyACM1, или какой-то другой. Выйти из tio: ctrl+T q

Важно, чтобы последовательный терминал был закрыт во время загрузки, потому что только одна программа может получить к порту доступ одновременно.

Печатать можно через printf(), но за счет того что он использует виртуальный файл stdout, как в Linux, он сильно менденнее, чем USBSerial.printf(), который пишет байты напрямую в USB
